# עיצוב: איש קשר ראשי = הלקוח (כברירת מחדל)

## מטרה
בדף לקוח, איש הקשר הראשי/ראשון צריך כברירת מחדל להיות הלקוח עצמו — בלי למלא שום פרט נוסף. רק אם המשתמש מציין שאיש הקשר שונה מהלקוח, נפתחים שדות למילוי פרטי אותו איש קשר.

זה חל **רק על איש הקשר הראשי**. רשימת "אנשי קשר" הקיימת (מודל הוספה/עריכה, `contacts` table, כמה אנשי קשר לכל לקוח) נשארת ללא שינוי — הפיצ'ר הזה מתווסף מעליה כמדור נפרד, ולא מחליף אותה.

## מודל נתונים
איש הקשר הראשי אינו נשמר כשורה בטבלת `contacts`. הוא נשמר כעמודות ישירות על `clients`, כי מדובר בערך יחיד לכל לקוח (לא רשימה):

```sql
alter table public.clients add column if not exists contact_is_self boolean not null default true;
alter table public.clients add column if not exists contact_name  text;
alter table public.clients add column if not exists contact_role  text;
alter table public.clients add column if not exists contact_phone text;
alter table public.clients add column if not exists contact_email text;
```

- `contact_is_self` — ברירת מחדל `true`. כל לקוח חדש/קיים/מיובא/נקלט-מליד מקבל זאת אוטומטית, בלי כל שינוי בקוד הייבוא/הקליטה.
- `contact_name/role/phone/email` — משמשים רק כש-`contact_is_self = false`. כש-`true`, הם עשויים להכיל ערכים ישנים (ראו "שימור נתונים" למטה) אך אינם מוצגים ואינם בשימוש.
- אין CHECK constraint שאוכף "name חובה כש-false" ברמת ה-DB — זה נאכף רק ב-UI (עקביות עם שאר הטופס בעמוד, שגם הוא לא אוכף חובה ברמת ה-DB).

## ממשק (ClientPage.jsx)
במדור "אנשי קשר" הקיים, מתווסף תת-מדור חדש **מעל** הרשימה הקיימת:

1. **כותרת המדור הקיים** משתנה מ"אנשי קשר (N)" ל-**"אנשי קשר נוספים (N)"** — כדי להבהיר את ההבחנה מאיש הקשר הראשי.
2. **כרטיס "איש קשר ראשי"** חדש, מעל הרשימה:
   - Checkbox: **"איש הקשר הוא הלקוח עצמו"** — משקף את `contact_is_self`, מסומן כברירת מחדל.
   - כשמסומן (`true`): מוצג תקציר קריאה-בלבד של שם/טלפון/אימייל **של הלקוח עצמו** (משדות `client.name` / `client.phone` / `client.email` — לא מהעמודות `contact_*`). אין עריכה כאן; לעריכת פרטי הלקוח הולכים ל"פרטי הלקוח" הרגיל בעמוד.
   - כשמבטלים את הסימון (`false`): התקציר נעלם, ובמקומו נפתחים 4 שדות inline (באותו סגנון `EditableField` כמו שאר הדף):
     - שם (חובה ב-UI, לא ב-DB)
     - תפקיד (אופציונלי)
     - טלפון (אופציונלי)
     - אימייל (אופציונלי)
   - כל שינוי בשדות נשמר מיידית דרך `patchClient` הקיים (`{ contact_name, contact_role, contact_phone, contact_email }`), באותו דפוס autosave-on-blur כמו שאר שדות הבסיס בעמוד.

## שימור נתונים בהחלפת מצב
כשחוזרים לסמן "איש הקשר הוא הלקוח עצמו" (`true`), הערכים ב-`contact_name/role/phone/email` **לא נמחקים** מה-DB — רק מוסתרים ב-UI. כך אם מישהו מבטל סימון בטעות ואז מחזיר אותו, המידע ששמר לא הולך לאיבוד. אם המשתמש רוצה למחוק את הפרטים בפועל, זה מחוץ לתחום הפיצ'ר הזה (אין כרגע כפתור "מחק פרטי איש קשר נפרד").

## מה לא משתנה
- מודל "לקוח חדש" (`handleCreate` ב-ClientsPage) — לא נוגעים בו; ברירת המחדל `contact_is_self=true` מגיעה מה-DB.
- `ImportClientsModal` / CSV import — לא נוגעים בו.
- `ingest_lead` RPC (קליטת לידים) — לא נוגעים בו.
- טבלת `contacts` ומודל ההוספה/עריכה שלה — ללא שינוי.
- מסך רשימת הלקוחות (`ClientsPage`, `ClientsTable`) — לא מוצג בו דבר חדש (מחוץ לתחום, YAGNI).
